# Sprint Guide

> **Typ:** Prozessreferenz

---

## Sprint-Status

| Status | Bedeutung |
|--------|-----------|
| BACKLOG | Geplant, noch nicht startbereit |
| BEREIT | Sprint-Doc ausgefüllt, keine Blocker |
| AKTIV | In Arbeit |
| PAUSIERT | Blockiert oder deprioritisiert |
| COMPLETE | Erledigt |

---

## Größen

| Größe | Dauer | Strategie |
|-------|-------|-----------|
| **S** | < 4h | Ein Commit am Ende |
| **M** | < 2 Tage | Commit pro Phase |
| **L/XL** | 3+ Tage | Feature-Branch, PR |

---

## Sprint-Lifecycle

1. `/sprint N "Name"` → Sprint-Dokument erstellen
2. Kontext, Ziel, Ansatz ausfüllen
3. Taskboard updaten
4. Implementieren mit Per-Step-Verifikation
5. Taskboard → COMPLETE, Learnings dokumentieren

---

## Anti-Patterns

| Anti-Pattern | Besser |
|---|---|
| Template nie ausgefüllt | Direkt beim Erstellen ausfüllen |
| Verifikation nur am Ende | Bash-Block nach jedem Schritt |
| Kein Rollback | Immer für M+ |
| Ziel = Maßnahme | Outcome formulieren |
